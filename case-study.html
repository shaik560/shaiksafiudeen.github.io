StudyCoach AI

An AI-powered learning assistant designed for Indian college students — bridging the gap between academics and career readiness with smart, personalized guidance.

The Problem

Indian college students face a major disconnect between academic learning and real-world career preparation. Existing tools are either too generic or too complex, leaving students overwhelmed and underprepared at the time they need it most.

StudyCoach AI was designed to solve this — an intelligent companion that understands where a student is and guides them to where they need to be, all within a single, intuitive interface.

Research & Discovery
  • Interviewed 12 college students across 3 disciplines
  • Identified top pain point: no personalized career guidance
  • Created 3 user personas based on behavioral patterns
  • Ran competitor analysis on 5 existing edtech platforms
  • Mapped user journeys to surface key friction points
Key Features Designed
  • Skill gap detection with visual progress mapping
  • Personalised career roadmap generator
  • AI resume scoring with actionable feedback
  • Mock interview simulator with prompts
  • Burnout-aware adaptive study planner
  • 6-screen hi-fi prototype in Figma
Design Process

I followed a double-diamond design process — starting broad with divergent research, then converging on core user needs. Lo-fi wireframes were sketched to validate layout before moving to hi-fi in Figma.

The visual language was kept clean and motivational — using a deep navy base with vibrant accent colours to inspire confidence without overwhelming students already dealing with academic stress.

Outcome & Learnings

All 5 usability test participants successfully completed core tasks with zero critical errors. Feedback highlighted the roadmap feature as the most valuable — students felt "seen" by the personalized guidance.

Key learning: designing for anxiety-prone users requires as much emotional design consideration as functional — clarity, encouragement, and visible progress are not just UX niceties, they are core to adoption.

Project Overview

Built during my internship, the Tourism Portal was designed to help travelers discover local destinations, attractions, and experiences through a clean and interactive web interface.

The goal was to create something that felt welcoming and easy to explore — a digital gateway to discovering places, not just a list of facts.

What Was Built

  • Integrated map-style navigation sections
  • Fully responsive layout — mobile to desktop

Technical Highlights
  • Pure HTML/CSS/JS — no frameworks

Outcome & Learnings

The project strengthened my understanding of building real, production-ready web pages from scratch without relying on libraries. It reinforced how good layout structure and thoughtful UI decisions make even a simple portal feel polished and trustworthy.

I gained confidence in responsive design patterns and how visual hierarchy guides users through content-heavy pages without overwhelming them.

Project Overview

MuCbe is a library web platform I designed and built as part of my role at Zuntra Digital. The client needed a modern, easy-to-navigate digital presence for their library service — one that felt welcoming to users of all ages and technical comfort levels.

The challenge was to balance visual appeal with functional clarity — making it easy for visitors to browse, search, and engage with the library's catalogue without friction.

Design Decisions
  • Clean, accessible layout for diverse age groups
  • Warm colour palette to evoke trust and comfort
  • Clear content hierarchy for fast scanning
  • Mobile-first responsive design approach
  • Intuitive navigation and CTA placement
Built With WixStudio

  • Dynamic content sections for catalogue display

  • Integrated client branding and typography

The site was delivered on time and received positive feedback from the client for its clarity and visual consistency. It demonstrated my ability to take a brief from concept through to a live, client-ready web product using WixStudio as the delivery platform.

Working within WixStudio's constraints while maintaining design quality was a valuable experience that sharpened my ability to adapt and problem-solve within platform limitations.

Random Number Game

An interactive browser-based guessing game that makes learning number logic and algorithm thinking fun — built with pure HTML, CSS, and JavaScript.

Project Overview

The Random Number Guessing Game is a browser-based interactive project built to sharpen JavaScript logic skills while creating something genuinely fun and engaging to play.

The game generates a random number and challenges the player to guess it within a set number of attempts — providing hot/cold feedback to guide the player toward the answer.

Features
  • Random number generation on each play
  • Attempt counter with win/lose logic
  • Higher / Lower feedback on each guess
  • Score tracking across rounds
  • Clean, minimal UI with instant feedback states
Technical Skills Applied
  • Vanilla JavaScript — no libraries used
  • DOM manipulation and event listeners
  • Conditional logic and game state management
  • CSS transitions for responsive feedback UI
  • Input validation and error handling
Outcome & Learnings

This project was a solid exercise in translating logic into interactive UI — making sure the game felt responsive and satisfying on every guess, win, or loss. It reinforced the value of instant, clear visual feedback in game design.

A simple concept executed well teaches more than a complex one executed poorly — and this project reminded me that constraints drive creativity.
